what is it

kelseywilliams.net is a lightweight api designed to transfer a small json string of data, date, a flag, and an id across the web with a simple request to https://kelseywilliams.net/api/handler.php. this api is perfect for small projects that don't want to buy web hosting, but still want to get some data to another local network. supports crud functions.

how to use

api key specifics

all requests should be made to https://kelseywilliams.net/api/handler.php

api key has a standard rate limit of 1 request every second. exceeding this number of requests will result in a status code of 429 Too Many Requests. as of right now, api keys are set to expire after 7 days allowing for a theoretical maximum of 604,800 requests.

request: GET

request must contain a key titled "key" with a value of the client's api key

returns JSON string and response code 200 Ok or a response code of 400 Bad Request if the key used is invalid

example: https://kelseywilliams.net/api/handler.php?key=some_api_key

request: POST

request must contain a message body with the keys "key", "data", "date", "flag" and "id" with values of the client's api key and four strings with a maximum character count of 16,777,215, 255, 255, and 16,777,215, respectively.

returns a response code of 201 Created or a response code of 200 Ok if the resource was not properly created or a response code of 400 Bad Request if the key used is invalid.

example in python's requests library: requests.post("https://kelseywilliams.net/api/handler.php", data = {"key" : "some_api_key", "data" : "Hello World!", "data" : "1/1/1970", "flag" : "false", "id" : "1234"})

request: PUT

request must contain a message body with the keys "key", "data", "date", "flag" and "id" with values of the client's api key and four strings with a maximum character count of 16,777,215, 255, 255, and 16,777,215 respectively.

returns a response code of 201 Created or a response code of 204 No Content if no resource was created or a response code of 400 Bad Request if the key used is invalid..

example in python's requests library: requests.put("https://kelseywilliams.net/api/handler.php", data = {"key" : "some_api_key", "data" : "Hello World!", "data" : "1/1/1970", "flag" : "false", "id" : "1234"})

request: DELETE

request must contain a message body with the key "key" with the value of the client's api key and an optional key of "id" with the value of a resource's id. if no id is provided, all rows on the table will be deleted. otherwise, only the resource whose id matches that specified in the request body will be deleted.

returns a response code of 204 No Content or a response code of 400 Bad Request if the key used is invalid.

example in python's requests library: requests.delete("https://kelseywilliams.net/api/handler.php", data = {"key" : "some_api_key", "id" : "1234"})
